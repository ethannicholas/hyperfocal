# Hyperfocal — working notes for Claude

Focus-stacking engine + macOS app. README.md explains the pipeline and
records shipped features; ROADMAP.md is the prioritized work list —
**completed work is removed from ROADMAP** (git history + README are the
record), and items are written self-contained so a cold session can pick
one up.

## Layout

- `Sources/HyperfocalKit/` — the engine (SwiftPM; CPU + Metal paths must
  stay bit-close: parity ≥ 90 dB).
- `Sources/hyperfocal-cli/`-equivalent target — CLI (`fuse`, `batch`,
  `synth`, `compare`).
- `App/` — sandboxed SwiftUI app. The Xcode project and `App/Info.plist`
  are **generated by XcodeGen from `App/project.yml`** (the `.xcodeproj`
  is gitignored); never hand-edit generated files, always `project.yml`,
  then `cd App && xcodegen generate`.
- `Probe/` — `retouch-probe` regression target. `AppModel.swift`,
  `Stack.swift`, `ProjectStore.swift`, `RetouchSession.swift` are
  **verbatim copies of App/Sources** compiled without AppKit UI. After
  editing any of those in App/Sources, `cp` them into `Probe/` or the
  probe drifts. `Probe/main.swift` is probe-only (extend it for new
  model-level regression checks — e.g. project-format round-trips).
- `Site/` — documentation site, hand-written static HTML/CSS, no build
  step; uploaded as-is to http://ethannicholas.com/hyperfocal.
- `Scripts/package-app.sh` — Developer ID build (version = git tag,
  build number = commit count); `Scripts/upload-mas.sh` — App Store
  Connect upload, same conventions.

## Invariants & gotchas

- `swift build && .build/debug/retouch-probe <synth frames…>` must print
  `probe: ALL PASS` before trusting model/engine changes; synth PSNR
  baselines live in ROADMAP's header.
- The app is sandboxed in every build. Save-panel grants cover exactly
  the chosen URL — temp/sibling writes near it are denied (see
  `ProjectStore.write` for the `.itemReplacementDirectory` pattern).
- SwiftUI `colorEffect` rasterizes in view-local coordinates: never
  attach shaders to views whose size scales with image dimensions
  (16,384 device-px texture limit, 65,504 half-float ceiling) — wrap a
  pane-sized clipped container instead (see PreviewPane).
- Working color space is Display P3 end to end; the tone curve
  (`ToneCurve`) must stay identical across its three consumers (preview
  shader, retouch canvas color cube, export CPU path).
- A failed save/export must never touch `AppModel.phase` — the fused
  result is still valid.
- Bundle ID `com.ethannicholas.hyperfocal` (lowercase — the capitalized
  form collides case-insensitively in Apple's App ID registry), team
  `Y3GFBT2WQ2`.
- UserDefaults suite `org.hyperfocal.settings` is deliberately decoupled
  from the bundle ID; renaming it orphans users' saved settings.

Use the project verify skill (`.claude/skills/verify/SKILL.md`): build +
launch recipes, AX automation quirks, synth fixtures, and the screenshot

running app, not just compiled.
